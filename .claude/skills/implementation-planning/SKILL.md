# 実装計画スキル

実装に入る前にコードを調査し、作業計画を立ててユーザーの承認を得るスキル。

## When to use

- 新機能の実装・リファクタリングを始める前
- issue が渡されたとき
- 実装範囲や方針を確定させたいとき
- 実装に対するフィードバックが与えられたとき

## Checklist / Instructions

- [ ] issue が渡されている場合は `gh issue view <issue番号>` で内容を確認する
- [ ] 関連するコードをコードベースで調査する（grep, glob, view ツールを活用）
- [ ] 実装方針・変更対象ファイル・影響範囲を整理する
- [ ] 不明点・曖昧な点があればユーザーに確認する
- [ ] 計画をユーザーに提示し、承認を得る
  - [ ] 承認を得たプランは `plan-<issue番号>-<概要タイトル>.md` というファイル名で保存し、実装の進捗とともに更新する
    - issue がない場合は `plan-<概要タイトル>.md` 形式とする
    - フィードバックがあたれらたときは、内容に追記していく
- [ ] issue の内容に不備があったり、ヒヤリングで発覚した事項があれば issue に新しくコメントを追加する

## Output Format

実装計画として以下を提示する:

1. **背景・目的**: 何を、なぜ実装するか
2. **実装方針**: どのように実装するか
3. **変更対象ファイル**: 作成・修正するファイルの一覧
4. **考慮事項・懸念点**: 影響範囲、注意点

## Dependencies

- `gh` コマンド（issue 参照時）
- コードベースの調査ツール（grep, glob, view）

## Notes

- 承認を得る前に実装を開始しないこと
- 不明点は遠慮せずユーザーに確認すること
