# rules 更新スキル

ユーザーからのフィードバックや新たなルールを `.claude/rules/*.md` に反映するスキル。

## When to use

- ユーザーからコーディングルールについてフィードバックがあったとき
- 繰り返し指摘される事項がある場合に再発防止のルールを追加するとき
- プロジェクト固有の慣習を文書化するとき

## Checklist / Instructions

- [ ] ユーザーの指摘内容がプロジェクト固有のものか、または間違えやすいものかを判断する
- [ ] 追加・更新する内容をユーザーに確認する（明示的に追加を指示された場合は確認不要）
- [ ] `.claude/rules/*.md` の適切なセクションを選択して更新する
- [ ] 更新内容をユーザーに報告する

## Dependencies

- `.claude/rules/*.md`

## Notes

- 汎用的な内容や自明なルールは追加しない
- プロジェクト固有のルールや、このコードベースで間違えやすい点を中心に記載する
- ユーザーに確認してから追加するのが原則（「このルールを追加してください」と明示された場合を除く）
