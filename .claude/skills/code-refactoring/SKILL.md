# リファクタリングスキル

既存の動作を維持しながらコードの品質を改善するスキル。

## When to use

- コードの可読性・保守性を改善したいとき
- 重複コードを排除したいとき
- 設計を改善したいとき

## Checklist / rules

- [ ] リファクタリング対象のコードを確認する
- [ ] 対応するテストが存在するか確認する
- [ ] テストがない場合は先にテストを書いて動作を保証する（`test-creating` スキルを参照）
- [ ] リファクタリングを実施する
- [ ] `pnpm test` でテストが引き続き通ることを確認する
- [ ] `pnpm lint` で lint エラーがないことを確認する
- [ ] 既存の動作が変わっていないことを確認する

## Dependencies

- `pnpm test` — テスト実行 (vitest)
- `pnpm lint` — lint チェック (biome)
- test-creating スキル

## Notes

- リファクタリングは機能変更を行わないこと
- 既存の動作に問題がありそうな場合はユーザーに確認する
- テストを先に書くことで動作の保証とリファクタリングの安全性を確保する
